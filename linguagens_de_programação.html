Linguagens de programação

Autor: Microlins Palhoça Data: 25/04/23

É perfeitamente compreensível ao iniciar no mundo da programação e informática haver dúvidas referentes a tipos de linguagens de programação. A utilização de jargões técnicos, pode não ser muito intuitivo para aqueles que iniciaram recentemente. Comumente palavras como, “Compilar, Run, Deploy, Commitar, legado” e tantas outras, podem deixar os menos familiarizados, um pouco confusos. Para isso tentarei de maneira simples e direta, explicar as diferenças entre algumas linguagens de programação e seus usos mais frequentes. Destaco que não há uma linguagem de programação melhor do que outra, mas sim linguagens de programação para objetivos diferentes, e que se adequem melhor em determinado cenário.

As linguagens de programação são divididas em vários grupos, porém, algo que comumente percebemos é que, em um grupo há as linguagens de scripts e no outro as linguagens compiladas, destaco que este fator não é um limitador, pois, linguagens de script também podem ser compiladas por um compilador.

As linguagens de script, são as que comumente são desenvolvidas através de scripts, com cada função ou procedimento sendo diretamente programado, e normalmente não possuem um ambiente de desenvolvimento gráfico, algo mais característico das linguagens compiladas. Um exemplo de linguagem de script é o JavaScript e o Python, ambas linguagens de script, comumente utilizadas em WebServices (Serviços da internet) e automatização.

As linguagens compiladas, se destacam principalmente por seus ambientes de desenvolvimento, que permitem um desenvolvimento rápido, dois ótimos exemplos são, C# e Delphi, linguagens que possuem ambientes de desenvolvimento rápido e integrado, sendo assim necessário compilar o código para que o mesmo funcione.

As linguagens de script são comumente mais aceitas por plataformas distintas, não sendo necessário a compilação do código para que a mesma funcione, em contra partida, linguagens compiladas, tendem a precisar de mais ajustes para funcionarem em outras plataformas, porém, tendem a um comportamento mais esperado, sendo assim mais práticas, para os desenvolvedores que buscam o formato final.

A melhor linguagem para o seu cenário depende de vários fatores, incluindo mas não restrito à esforço computacional, tempo de desenvolvimento, comportamento multiplataforma etc.. Lembra dos termos que citei no começo deste artigo? Os jargões técnicos citados, tem significados, o termo “Compilar” vem de “Juntar ou preparar” sendo muito utilizada em linguagens compiladas, que precisam ser juntadas e preparadas para o usuário final. Já o termo “Run” vem de “Executar” normalmente associada a execução tanto para testes quanto para execução do script.

Quanto aos termos “Commitar e Deploy” vem dos termos “Comentar e Aplicar” muito utilizado em arquiteturas do Git, para comentar uma versão ou aplicar as alterações em um determinado código ou serviço.
